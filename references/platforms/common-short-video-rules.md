# 通用短视频规则

所有平台都先经过这一层，再进入平台专项检查。

## 一、检查目标

输入待发布内容后，按以下顺序判断：

```text
输入内容
→ 平台规则检查
→ 内容质量检查
→ 风险检查
→ 通过 / 修改后发布 / 不建议发布
```

输出必须同时包含：

- 综合得分；
- 发布结论；
- 严重问题；
- 一般问题；
- 建议修改；
- 可直接交付的替代文本。

## 二、四档发布结论

1. **可以直接发布**：没有严重问题，只有轻微优化空间。
2. **小改后发布**：核心方向对，但存在 1—3 个会影响传播或转化的问题。
3. **建议重写**：内容目标、结构或平台适配明显错误，继续小修意义不大。
4. **不建议发布**：存在严重失真、抄袭、违规、强误导或与用户身份强冲突的问题。

## 三、通用评分维度

默认总分 100 分。平台专项维度可以二次加权，但通用层先完成这 8 项：

1. **目标清晰**：开头是否能看出讲给谁、解决什么问题。
2. **钩子有效**：前几秒是否能建立冲突、结果差、损失感、身份认同或好奇缺口。
3. **承诺兑现**：正文是否真的回应开头，而不是标题和正文脱节。
4. **信息密度**：是否有重复、空话、铺垫过长、逻辑跳跃。
5. **证据可信**：案例、数据、截图、体验、过程是否可解释、可追问、不过度夸大。
6. **表达可拍**：文案能否落到画面、镜头、字幕、口播和剪辑动作。
7. **商业自然**：产品、服务或行动号召是否从正文自然延伸，而不是突然插入。
8. **合规与原创**：是否存在违规承诺、敏感风险、明显套壳、过度借鉴或版权问题。

## 四、严重问题判定

出现以下任一情况，默认不能给出“可以直接发布”：

- 明显看不出目标用户；
- 开头承诺与正文内容不一致；
- 关键事实、案例或数据无法自圆其说；
- 产品露出过早，广告感压过内容价值；
- 明显照搬对标结构且只替换名词；
- 含绝对化收益承诺、医疗金融高风险表达或灰色引导；
- 结尾 CTA 混乱，同时推动多个动作。

## 五、内容质量检查

重点检查：

- 是否有一个明确主问题，而不是三四个问题堆在一起；
- 段落之间是否有推进关系，而不是想到哪说到哪；
- 每一段是否能用一句话概括；
- 每一个重要结论前是否有铺垫证据；
- 结尾是否推动一个最合理的下一步。

## 六、风险检查

至少检查以下风险：

- 违规承诺风险；
- 误导性表述风险；
- 版权或搬运风险；
- 平台风格失配风险；
- 用户真实身份承接不足风险；
- 发布后评论区失控风险。

## 七、默认输出格式

```markdown
# 发布前检查

平台：<平台名>
综合得分：82/100
发布结论：小改后发布

## 严重问题
1. ...

## 一般问题
1. ...

## 建议修改
- 新开头
- 完整优化脚本
- 封面短标题
- 发布配文
- 置顶评论
```
